Introduction to Python for Consultants

Learn some fundamentals in order to integrate Python into your day to day workflow

Purpose

This tutorial aims to provide the typical Consultant and Senior Consultant with a foundation to begin using Python with purpose. The operative word is begin; we want people to start thinking about how Python can solve problems, create efficiencies, and ultimately drive value for clients. This guide will get the user from absolutely no knowledge of Python to a point where they can take it upon themselves to continue developing this skill. There will be a mix of step-by-step instructions and actual code that you can start using immediately. After this, you should have enough context to experiment and learn more when non-billable.

Install Python (Miniconda)

The set up process should be the same on both your AWS and local desktops (save the drive location). As such, we recommend going through this process for both systems. Having the ability to code in either environment is very useful. For example, Python and SQL can only be interoperable in the AWS environment. In other cases, it can be a good idea to keep a copy of a client project locally to isolate code from the primary folder.

We will use an open-source distribution of Python called Miniconda. It is a stripped down version of the widely popular Anaconda Data Science Platform. It provides tools for managing virtual environments and package dependencies, emphasizing reproducability and ease of use. Go ahead and download Miniconda from their website, here. There are several options, but choose the Python 3.x installer for the 64-bit version of Windows.


Run the installer and go through the following steps:

  1. Press Next and agree to the License Agreement.
  2. Install locally ("Just Me").

    3. IMPORTANT:
  3. Change the Destination Folder to either:
    • On local desktop: C:\Miniconda3
    • On AWS desktop: D:\Miniconda3

  4. Press Next and make sure both Advanced Options are not selected.
  5. Proceed with the installation (refer to images below)


  6. Run the script called verify_install.bat
    • The installation is successful if a Python console opens
    • At the top of the console, it should say the Python version number and "Continuum Analytics"
    • Type: print("Hello, World!")
    • Congratualtions, you've written Python program!
  7. Finally, run the script called create_env.bat
    • This may take a minute, but it is creating a special Python environment by installing several packages that are useful to common workflows in Python. To read more about Conda Environments, refer to their docs here.


Install PyCharm IDE
  1. First, download the Community version of PyCharm here. Simply go through default settings and run it once finished.
    • While the default settings are fine, feel free to select the option that associates all *.py files with PyCharm.
    • If PyCharm asks to import settings, just tell it to not import and press "OK" to complete the installation process.
    • It could also ask to set a theme; choose either and note it can be changed at any time.
    • Next, it may ask to install featured plugins. It doesn't hurt to install plugins for Markdown, Bash, and R although we will mostly ignore them.

  2. Now select Create New Project. PyCharm will point to a new folder called PyCharmProjects under your home directory.
    • Just change the untitled portion of the Location's filepath to Tutorial. This will create a project folder that we can use moving forward.

    • Note that every time you create a new Python project in PyCharm, you will be given a similar dialogue option. While you do not have to keep all project files in the default PyCharm directory, it is a simple way to organize projects.
    • Another point is that PyCharm will always open to the last Project that you were working on.

  3. Now we need to point PyCharm to the correct Python interpreter. This was created in the last section. Activate the Project Interpreter dropdown and select Existing Interpreter. Then select the gear icon and click on Add Local...
    • Click on the three dots and navigate to the Python executable in the path shown below. Also, make this interpreter available to other projects as a convenience.


  4. Welcome to the PyCharm IDE! A few things to note:
    • The window to the left is a file explorer for your project directory. It is very useful when jumping between project files and can be used to view flat files and other file types other than Python scripts.
    • PyCharm is a keyboard centric program - much like Excel hotkeys, you can learn shortcuts that make using PyCharm more productive.
    • To start off, go to Settings with "Ctrl + Alt + S" (File -> Settings)
      • Go to Project -> Project Interpreter to view which Python executable is being used and a list of installed packages. This is should already be set but it is handy to know where it can be changed.
      • If you are a hotkey maniac, then you might want to spend some time looking at the Keymap section in settings. It has all default shortcuts and every function can be remapped however you like!
    • Finally, one of the most useful aspects of using PyCharm is having an integrated Python Console. To pin it to your screen, go to Tools -> Python Console...

  5. The last step is to write and run our first Python program.
    • Create a new Python file with "Alt + Insert + Enter" or right click on the Project folder in the explorer. Name the file hello.py.
    • The fastest way to run this program is to right click on the file and select "Run..."
    • Another way to run Python code, which is akin to using SQL Studio, is to highlight a block of code and press "Alt + Shift + E" or right click and select "Execute Selection in Console"
      • Pro-tip: re-map the "Execute Selection in Console" hotkey to "Alt + X" as to match the hotkey in SQL Studio.
      • While running blocks of code is both fast and familiar, it is best practise to write scripts such that they can be run at the "module-level" - which is to say one can simply run the entire file in one go.
